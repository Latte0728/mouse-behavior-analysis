# 프로젝트 공통 가이드라인 (동물 행동 분석 AI 플랫폼)

본 규칙은 해당 프로젝트(c:\my_project)에서 Streamlit 기반의 대시보드를 개발하거나 수정할 때 항상 최우선으로 지켜야 하는 핵심 메모리(Memory) 및 UI 패턴입니다.

## 1. 전반적인 UI/UX 및 디자인 규칙
- **이모지 사용 절대 금지:** 전문적인 B2B SaaS 느낌을 유지하기 위해 이모지(😊, 📊 등)는 절대 사용하지 않습니다.
- **순수 SVG 아이콘 활용:** 이모지나 외부 이미지 대신, 언제나 고해상도를 유지할 수 있는 순수 SVG 라인 아트(Line-art) 코드를 `st.markdown(..., unsafe_allow_html=True)`를 통해 직접 삽입합니다. (예: 대시보드의 플라스크, 마우스, 체크보드, 문서 아이콘 및 버튼 아이콘)
- **핵심 브랜드 컬러:** 메인 포인트 컬러로 파란색(`#2563eb`)을 사용하며, 보조 텍스트 컬러는 회색(`#94a3b8`, `#64748b`)을 사용해 일관성을 맞춥니다.

## 2. Streamlit Multi-Page App (MPA) 구조
- `app.py`를 메인 엔트리 포인트로 두고 `st.navigation`과 `st.Page`를 활용해 사이드바 메뉴를 구성합니다.
- 실제 페이지 로직은 `pages/` 폴더 내에 분리하여 작성합니다 (`01_dashboard.py`, `02_experiments.py` 등).
- 각 페이지 이동 간 부자연스러운 새로고침을 방지하고 상태(Session State)를 자연스럽게 유지합니다.

## 3. 커스텀 컴포넌트 패턴
- **마법사(Wizard) UI / Stepper:** `04_comparison.py`에 적용된 화살표 꼬리표 모양의 CSS Stepper 로직을 표준으로 사용합니다. (`.stepper-wrap`, `.step` CSS 클래스 재사용)
- **테이블(표) 렌더링:** `st.dataframe` 기본 UI의 시각적 한계(뱃지 삽입 불가, 커스텀 아이콘 삽입 제한)를 극복해야 할 때는, pandas DataFrame을 `df.to_html(escape=False)`로 변환한 뒤 커스텀 CSS(`.custom-table`)를 입혀 렌더링합니다.
- **상태 배지(Status Pill):** '진행 중', '완료' 등의 상태 텍스트는 테두리가 둥근 Pill 형태의 배경색 뱃지(HTML/CSS)로 감싸 시각적 구분을 명확히 합니다.

## 4. 데이터 시각화
- 차트 렌더링 시에는 Altair, Plotly 등 반응형 차트 라이브러리를 적극 활용하여 다크모드/라이트모드 및 컨테이너 너비에 맞게 유동적으로 표시되도록 합니다.
